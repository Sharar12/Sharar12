## 🚀 Quick Start

### Prerequisites
- Node.js 18+
- PHP 8.2+
- MySQL 8.0+
- Composer

### Backend
```bash
cd backend
composer install
cp .env.example .env       # then set DB_*, APP_KEY
php artisan key:generate
php artisan migrate --seed
php artisan serve
```

### Frontend
```bash
cd frontend
npm install
cp .env.example .env.local # set NEXT_PUBLIC_API_URL
npm run dev
```

Visit `http://localhost:3000` 🎉

## 5. The Repo Selection Rubric

You said you have **lots of small experiments**. Pin at most **6**, but only the ones that pass this rubric. Score each repo 0–3 on each dimension. **Pin anything ≥ 12 / 18.**

| Dimension | 0 | 1 | 2 | 3 |
|-----------|---|---|---|---|
| **Has a real README?** | empty | one paragraph | decent | polished + screenshots |
| **Does the thing actually run?** | broken | runs on my machine | runs from a clean clone | has CI passing |
| **Evidence of completion** | abandoned | partial | done, no docs | done + documented + tested |
| **Code quality** | <3 commits, messy | ok | clean | clean + comments + tested |
| **Demonstrates a skill you want hired for** | gimmick | tangentially | yes | headline piece |
| **Public-facing or showable** | private | private + ugly readme | public, readable | public + live demo |

**Apply this to your repos, then:**

- Pin the **top 3** for sure.
- Pin 2–3 more **only** if you're confident.
- **Archive (don't pin)** anything that scores <9/18. Archiving is allowed — it shows judgment.
